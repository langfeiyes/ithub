# 二、IO入门

# 2.1 File类（掌握）

File可以理解为文件和文件夹（目录），用于表示磁盘中某个文件或文件夹的路径。主要用于文件和目录的创建、查询和删除等操作。

注意：

  • 只能获取和设置文件本身的信息（文件大小，是否可读），不能设置和获取文件里面的内容。
  • 路径分隔符
    • Unix/Linux/Mac: 使用/来表示路径分隔符。
    • Windows: 默认情况下是不区分大小写的，使用\来分割目录路径。但是在Java中一个\表示转义，所以在Windows系统中就得使用两个\\。

构造器

  • File (String pathname) 根据给定的路径名来创建一个File对象

常用方法 （请使用ChatGPT完成）

File类的创建功能

  • boolean createNewFile() : 创建一个新的空的文件
  • boolean mkdirs()：创建当前目录和上级目录

File类的删除功能

  • boolean delete() ：删除文件或者目录

File类的获取功能

  • String getName()：获取文件名称
  • String getPath()：获取文件的路径
  • File getParentFile()：获取上级目录
  • String getAbsolutePath()：获取绝对路径

File类的判断功能

  • boolean exists()：判断文件或者目录是否存在
  • boolean isFile() ：是否是文件
  • boolean isDirectory()：判断是否是目录
public class FileDemo {
	public static void main(String[] args) throws Exception {
		File f = new File("C:/test/123.txt");
		System.out.println(f.getName());//123.txt
		System.out.println(f.getPath());//C:/test/123.txt
		System.out.println(f.getAbsolutePath());//C:/test/123.txt
		System.out.println(f.getParentFile().getName());//test
		System.out.println(f.exists());//true
		System.out.println(f.isFile());//true
		System.out.println(f.isDirectory());//false
		
		//如果当前文件的父文件夹不存在,则创建
		if(!f.getParentFile().exists()) {
			f.getParentFile().mkdirs();
		}
	}
}

# 2.2 字符编码

# 2.2.1 编码表介绍 ( 了解 )

  • 计算机中储存的信息都是用二进制数据表示的；我们在屏幕上看到的英文、汉字等字符是二进制数转换之后的结果

  • 按照编码表规则，将字符存储到计算机中，称为编码。

  • 按照同样的编码表规则，将存储在计算机中的二进制数据解析显示出来，称为解码 。

  • # 简单理解：
    • 存储一个字符a，首先需在码表中查到对应的数字是97，然后按照转换成二进制的规则进行存储，称为编码
    • 读取的时候，先把二进制解析出来，再转成97，通过97查找编码表中对应的字符是a，称为解码
  • # ASCII码表：
    • 包括了数字字符，英文大小写字符和一些常见的标点符号字符。
    • 注意：ASCII码表中是没有中文的。
  • # GB2312/GBK/GB18030：
    • win7系统默认的码表。兼容ASCII码表，也包含了21003个汉字，并支持简、繁体汉字
    • 注意：GBK是中国的码表，一个中文以两个字节的形式存储。但不包含世界上其他国家的文字。
  • # Unicode码表：
    • 由国际组织ISO 制定，是统一的万国码表，计算机科学领域里的一项业界标准，容纳全球多数国家的常见文字和符号。
    • 注意： Unicode是万国码表标准，以UTF-8实现后一个中文以三个字节的形式存储

存储字母、数字、汉字的常识 （记住） ：

（1）存储字母和数字无论是什么字符集都占1个字节.

（2）存储汉字，GBK家族占两个字节，UTF-8家族占3个字节。

注意：不能使用单字节的字符集（ASCII）来存储中文，否则会乱码。

# 2.2.2 字符串的编码和解码操作（知道）

  • 编码：把字符 按照编码表 转成二进制数据
    • byte[] string.getBytes("UTF-8")
  • 解码：把二进制数据 按照编码表 转成字符
    • new String(bytes, "UTF-8")

注意：编码和解码使用的码表必须一致，否则会导致乱码。

public class EncodeDemo {
    public static void main(String[] args) throws UnsupportedEncodingException {
        // 字符编码: 把字符串 按照指定的字符集转成 二进制序列(byte数组) => getBytes()
        String str = "abc中";
        byte[] bytes = str.getBytes("UTF-8");
        System.out.println(Arrays.toString(bytes));

        // 字符解码： 把二进制序列(byte[]) 按照指定的字符集转成字符
        String str2 = new String(bytes, "GBK");
        System.out.println("str2 = " + str2);
    }
}

# 三、IO流介绍

# 3.1 为什么要学习IO流

  • 通过变量,数组,或者集合存储数据
    • 都是不能永久化存储 , 因为数据都是存储在内存中
    • 只要代码运行结束，所有数据都会丢失
  • 使用IO流
    • 将数据写到文件中，实现数据永久化存储
    • 把文件中的数据读取到内存中(Java程序)

# 3.2 IO操作示意图

  • I 表示input ，是数据从硬盘进内存的过程，称之为读。
  • O 表示output ，是数据从内存到硬盘的过程。称之为写
  • IO的数据传输，可以看做是一种数据的流动，按照流动的方向，以内存为参照物，进行读写操作

讲解IO知识点的时候，习惯和生活中的水流联系起来，一起来看看复古的水井和水缸。

此时站在水缸的角度，分析IO的操作方向：

输入操作：水井——>水缸

输出操作：水缸——>饭锅

也就是说：谁拥有数据，谁就是源，把数据流到哪里，哪里就是目的地。那么，请问水缸是源还是目标？

我们一般**站在程序( 内存 )**的角度：

  • 程序需要数据 → 把数据读进来 → 输入操作（read）：读进来

  • 程序保存数据 → 把数据写出去 → 输出操作（write）：写出去

六字心决：读进来，写出去（仔细揣摩这六个字有什么高深的含义）

# 3.3 流的分类（掌握）

根据流的不同特性，流的划分是不一样的，一般按照如下情况来考虑：

  • 按流动方向：分为输入流和输出流。（写程序时，一定要确定角度）
  • 按数据传输单位：分为字节流和字符流，即每次传递一个字节（byte）或一个字符（char）
    • 字符和字节是什么关系？
    • 一个字符按照编码表可以编码成一个或者多个字节
  • 按功能上划分：分为节点流和处理流，节点流功能单一，处理流功能更强 。

# 四、字节流

# 4.1 文件字节输出流 ( 重点 )

# 4.1.1 文件字节输出流介绍

  • # FileOutputStream类 :
    • OutputStream 字节输出流最顶层的类 , 抽象类
    • FileOutputStream 类是文件字节输出流，用于将数据写出去到文件

# 4.1.2 FileOutputStream 成员

  • public FileOutputStream(File file)：创建文件输出流以写入由指定的 File对象表示的文件。
  • void write(int b) 一次写一个字节
  • void write(byte[] buff) 一次写多个字节

操作步骤

  • 1 确定源和目的地

  • 2 创建管道（创建字节输出流对象）

  • 3 把数据写出去

  • 4 释放资源

需求1：使用文件字节输出流，把程序中数据保存到a.txt文件(先只操作英文)

public class FileOutputStreamDemo {
    public static void main(String[] args) throws IOException {

        // 1 确定数据源(程序内存)和目的地(a.txt)
        String str = "hello";
        File file = new File("G:\\test\\a.txt");

        // 2 建立管道（通过具体的IO流对象创建管道）
        FileOutputStream out = new FileOutputStream(file);

        // 3 把数据写出去
        // 写到文件中就是以字节形式存在的，只是文件帮我们把字节翻译成了对应的字符 , 方便查看
        /*out.write(97);
        out.write(98);
        out.write(99);*/

        // 一次写入多个字节
        byte[] buf = str.getBytes("UTF-8");
        out.write(buf);

        // 4 释放资源
        out.close();
    }
}

# 4.2、文件字节输入流 ( 重点 )

# 4.2.1 字节输入流介绍

  • InputStream类 : 字节输入流最顶层的类 , 抽象类
  • FileInputStream类 : 文件字节输入流，专门用于读文件

# 4.2.2 FileInputStream 类成员

  • public FileInputStream(File file) : 从file类型的路径中读取数据
  • int read() 一次读一个字节，读取到末尾返回-1
  • int read(byte[] buff) 一次读多个字节
    • 从输入流读取最多b.length个字节的数据
    • 返回的是真实读到的数据个数

操作步骤

  • 1 确定源和目的地
  • 2 创建管道（创建文件字节输入流对象）
  • 3 把数据读进来
  • 4 释放资源

需求1：使用文件字节输入流，读取b.txt文件中的数据

public class FileInputStreamDemo {
    public static void main(String[] args) throws IOException {
		// 1 确定源(a.txt)和目的地(程序内存)
        File file = new File("G:\\test\\b.txt");
        // 2 建立输入流管道( 创建具体的IO流对象 )
        FileInputStream in = new FileInputStream(file);
        
        // 3 把数据读进来:一次读取多个字节
        byte[] buf = new byte[2]; 	// 临时数组
        int len; 					// 表示读取的有效字节的个数
        
        /** 过程解析: 
         * 1 buff和len用于存储读进来的字节和有效个数
         * 2 len=in.read(buff) 读进来多个字节存入buff中，有效个数存入len中
         * 3 把buff中有效的字节，解码成字符，new String(buff,0,len,"UTf-8")
         */
        while( (len = in.read(buf)) != -1 ) {
             System.out.print(new String(buf , 0 , len));
        }

        // step 4: 关闭管道
        in.close();
    }
}

# 5.2.2 FileReader的成员

字节流和字符流选用问题 ( 面试题 )

  • 使用记事本打开某个文件，如果可以看到内容的就是文本文件，否则可以暂时认为是二进制格式的。

  • 一般的，操作二进制文件（图片、音频、视频等）必须使用字节流。操作文本文件使用字符流，尤其是操作带有中文的文件，使用字符流不容易导致乱码，因为使用字节流可能出现读取半个汉字的尴尬（汉字由两个或三个字节组成）。当然，如果不清楚属于哪一类型文件，都可以使用字节流。

综合应用：把copy_before.txt文件中的数据拷贝到copy_after.txt文件中

private static void copy() throws Exception {
		// 1 确定源和目标
        File fromFile = new File("F:\\test\\copy_before.txt");
        File toFile = new File("copy_after.txt");

        // 2 创建输入流和输出流管道
        Reader reader = new FileReader(fromFile);
        Writer writer = new FileWriter(toFile);

        // 3 具体的IO操作
        // 每次读取1024个字符
        char[] cbuff = new char[1024];
        int len;
        while ((len = reader.read(cbuff)) != -1) {
            writer.write(cbuff,0,len);
        }

        // 4 关闭资源
        reader.close();
        writer.close();

        System.out.println("复制完成");
}

# 六、包装流

前面讲的文件流( FileInputStream/FileOutputStream 、FileReader/FileWriter )属于节点流，节点流的功能都比较单一，性能较低。

我们能否对节点流进行包装，并提供基于节点流并比它更强大的功能呢？

包装流，相对于节点流更高级，功能更强大，这里存在一个设计模式——装饰者设计模式，此时撇开不谈。

包装流如何区分？写代码的时候，发现创建包装流对象时，需要传递另一个流对象，类似：

new  流类B(  new 流类A(..) ) ;
1

那么流B就属于包装流，当然A可能属于节点流也可能属于包装流。

有了包装流之后，我们只关心包装流的操作即可，比如只需要关闭包装流即可，无需在关闭节点流。

非常重要的包装流——缓冲(Buffer)流，刚才字节流和字符流都有各自的包装流：

BufferedInputStream / BufferedOutputStream 字节缓冲流
BufferedReader / BufferedWriter 字符缓冲流
1
2

缓冲流特点：

  • 内置了一个默认大小为8192个字节或字符的缓冲区
  • 缓冲区的作用用来减少磁盘的IO操作，拿字节缓冲流举例，比如一次性读取8192个字节到内存中，或者存满8192个字节再输出到磁盘中。
  • 操作数据量比较大的流，都建议使用上对应的缓冲流。

# 6.1 字节缓冲流

需求：把郭德纲-报菜名.mp3文件中的数据拷贝到郭德纲-报菜名2.mp3文件中

private static void copy3() throws Exception {

        File srcFile = new File("郭德纲-报菜名.mp3");
        File toFile = new File("郭德纲-报菜名2.mp3");

        // 文件输入流管道
        FileInputStream in = new FileInputStream(srcFile);
        BufferedInputStream bis = new BufferedInputStream(in,8192);

        // 文件输出流管道
        FileOutputStream out = new FileOutputStream(toFile);
        BufferedOutputStream bos = new BufferedOutputStream(out,8192);

        byte[] buf = new byte[1024];    // 每次可以读取1024个字节(1k)
        int len = -1;                   // 读取了多少个有效字节
        while ( (len = bis.read(buf)) != -1 ) {
            // 把每次读的1k缓存起来，到8k后自动一次性写出去
            bos.write(buf,0,len);
        }

        bos.close();
        bis.close();
}

# 6.2 字符缓冲流

  • readLine() : 一次读取一行，读到末尾返回null

  • newLine() : 写入一个换行。
