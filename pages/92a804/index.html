# 2.1.1 什么是数组(理解）

简单地说，数组是一个容器，用来存储相同类型数据的容器

所谓数组(Array)，是一种容器，里面存储相同类型数据，这些数据按一定编号排列存储在这个容器中。

  • 这些类型相同的多个数据需要一个名字以便统一管理它们，这个名字称为数组名
  • 组成数组的各个数据称为数组的元素( element )，数组中元素的个数称为数组的长度(length)。
  • 数组每个元素都用一个唯一的编号称为下标或索引(index)。表示元素存放的位置，索引从0开始，步长是1，有点像Excel表格的行号。

数组的特点：

1、数组的长度一旦确定就不能修改

2、创建数组时会在内存中开辟一整块连续的空间，每个空间都已被编号

3、存取元素的速度极快，因为可以使用下标，直接定位到任意一个元素。

# 2.2 数组的初始化（重点）

数组在定义后，必须初始化才能使用。所谓初始化，就是在内存中给数组分配存储空间，并为每一个元素赋上初始值，有两种方式：

  • 静态初始化；

  • 动态初始化；

数组的长度是固定的，无论以哪种，一旦初始化完成，数组的长度（元素的个数）就固定了，不能改变，除非重新对它初始化。

那么静态初始化和动态初始化如何选用呢？

  • 如果我们事先知道元素是什么，选用静态初始化；
  • 事先不知道元素是什么(但知道需要多少个的空间)，选用动态初始化。

# 2.2.1 静态初始化（重点）

程序员直接为每一个数组元素设置初始值，而数组的长度由元素个数决定。

# 4.2 求最大值和最小值

需求2：求出int类型数组中最大元素值

思路：

  1. 暂定第一个元素为最大，用变量max存起来
  2. 用max和后面的元素比较，如果元素比max大，则把这个元素赋值给max
  3. 遍历完成后，max存的数值就是最大值
public class ArrayDemo9 {
	public static void main(String[] args) {
		int[] nums = new int[] { 11, 22, 33, 44, 22, 55 };

            // step 3: 取出来的元素和max比较，如果取出来的元素大于max，把大的元素存入max中，直到全部比较完成

# 4.3 统计和累加求和

需求：某部门5名员工的销售额分别是：16、128、36、6、100，完成以下需求：

  • 计算出他们部门的总销售额
  • 达标销售额为100，统计完成销售额的人数
public class ArrayDemo10 {
    public static void main(String[] args) {
        int [] arr = {16,128,36,6,100};
        // (1)计算出他们部门的总销售额
        int total = 0;
        // 1 遍历数组，取出每个元素
        for (int ele : arr) {
            // 2 定义变量total，累加每个元素
            total += ele;
        }
        System.out.println("total = " + total);

        // (2)达标销售额为100，统计完成销售额的人数
        int count = 0;
        for (int ele : arr) {
            if(ele >= 100) {
                count++;
            }
        }
        System.out.println("count = " + count);
    }
}

# 4.4 格式化输出数组元素

需求：按照某种格式打印数组元素，数组元素放在方括号[]中,相邻元素使用逗号分隔开。如上述数组打印出来，效果如：[11, 22, 33, 44, 22, 55]

思路：

  1. 定义一个String类型的变量，准备用来拼接指定格式的字符串
  2. 将 [ 拼接到字符串中
  3. 通过循环遍历的方式，将每个元素拼接到字符串中，每个元素后面拼接一个逗号，
  4. 如果是最后一个元素，不是拼接逗号，而是拼接一个 ]
public class ArrayDemo11 {
	public static void main(String[] args) {
		int[] nums = new int[] { 11, 22, 33, 44, 22, 55 };
        // str表示结果字符串,先拼一个[符号
		String str = "[";
		for (int i = 0; i < nums.length; i++) {
            int ele = nums[i];
			// 把每一个元素拼接在str后面 
			str = str + ele;
			// 如果是最后一个元素,拼接]
			if(i == nums.length-1) {
				str = str + "]";
			}else {
				// 如果不是最后一个元素拼接,
				str = str + ", ";
			}
		}
		System.out.println(str);
	}
}

# 5.1 JVM内存模型（了解）面试题

JVM内存划分，人为的根据不同内存空间的存储特点以及存储的数据。

  • 程序计数器：当前线程所执行的字节码的行号指示器。

  • 本地方法栈：为虚拟机使用的native方法服务。

  • 方法区：存储已被虚拟机加载的类 (xx.class ) 信息

  • **Java虚拟机栈：**简称栈区（stack），每当调用一个方法时，在栈区分配方法执行时的内存，存放了当前方法的变量，当方法调用完毕，该方法的内存就被销毁了。一句话：java方法执行时，在栈区执行

  • **Java堆：**简称堆区(heap), 所有的对象以及 数组 都要在堆上分配内存。每次使用new关键字，就表示在堆内存中开辟一块新的存储空间。

  • GC(Garbage Collection)，垃圾回收器。Java的自动垃圾回收机制可以简单理解为，不需要程序员手动的去控制内存的释放。当JVM内存资源不够用的时候，就会自动地去清理堆中无用对象（没有被引用到的对象）所占用的内存空间。

// jvm 内存模型
/**

* 3 堆区：数组的内存都分配在该区。该区变量有一个标志——new操作符，表示在堆内存中开辟一块新的存储空间。

对上述代码做内存分析

int[] nums = new int[] { 1, 3, 5, 7 };

这一行代码可以分成三步：

  • 在堆内存中开辟一块空间，用来存储数组数据:new int[] { 1, 3, 5, 7 }

  • 在栈中开辟一块空间nums

  • 把堆空间表示数组的内存地址赋给nums

这种把内存地址赋给一个变量，也被称之为引用关系（为了更清晰有人习惯画一根箭头来表示这种关系），也就是说nums变量引用了堆中某一块内存地址，当操作nums变量的时候，其实底层操作的是nums所引用内存地址里面的数据。

所以此时，通过nums.length代码来查看nums数组中有几个元素，结果很明显是4。

nums = new int[] { 2, 4, 8 };
1
  • 因为存在new，说明又会在堆空间开辟一块新的空间，赋初始值。

  • 并把内存地址重新赋给nums变量，nums原来所引用的地址将被覆盖掉。

所以此时，通过nums.length代码来查看nums数组中有几个元素，结果很明显是3。

注意：此时地址为0x1234的内存空间没有被任何变量所引用，所以该空间就变成了无用的垃圾，就等着垃圾回收器回收该空间。

如果存在一行代码，如下：

nums = null;

null表示不再引用堆中的内存空间，那么此时nums就好比是没有初始化的，不能使用。
