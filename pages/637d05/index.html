# 二、接口

# 2.1 接口概述（了解）

接口 : 如果一个类中都是抽象方法 , 那么这个类应该是定义规则的类 , 我们应该把此类定义成接口,接口是一种引用数据类型

  • 作用 :
    • 用于定义规则 。接口中的方法都是抽象方法，并不提供功能实现（implement）
    • 增强程序的扩展性。接口体现了规则和实现相分离的思想(这种向分离的思想就可以增强程序的可拓展性)
电脑的显卡分为集成显卡和独立显卡：
- 集成显卡：显卡和主板焊死在一起，显卡坏了，只能换主板
- 独立显卡(有卡槽)：显卡和主板相分离，显卡插到主板上即可，显卡坏了，只换显卡，不用换主板

# 2.3 接口的实现类（重点掌握）

接口用于定义（声明）规则，那么总该有人遵守这套规则，否则接口没什么用呀？接口要有实现类才有意义

此时必须定义一个类去实现接口，并覆盖接口中的方法，这个类称之为实现类，实现类和接口之间的关系称之为实现关系，使用implements 关键字

public class 类名 implements 接口名 {
    
}
1
2
3

实现类特点：必须实现接口中的所有抽象方法，完成功能代码

此时接口和实现类之间的关系：

  • 接口：定义多个抽象方法，仅仅定义有哪些功能，却不提供实现。
  • 实现类：实现接口
    • 要么实现接口中所有抽象方法，完成功能具体的实现。
    • 要么也是一个抽象类

如果实现类没有全部实现接口中的方法（下图）

# 2.4 接口总结

  • 接口用于定义规则，它里面规定了一系列功能（抽象方法），它可以被多个类实现 (遵守)。

  • 实现类实现接口，必须实现接口中的所有抽象方法。我们经常说：接口规定了实现类应该具备的功能。

# 三、多态（掌握）

# 3.1 多态概念

多态时面向对象三大特征：封装、继承、多态。

在继承关系，是一种 is A 的关系，也即什么是什么,也就说子类是父类的一种特殊情况，有如下代码：

例如：如果Student 继承于 Person，我们就可以说，学生 is a 人类

public class Animal { }
public class Dog extends Animal{ }   // Dog is a Animal
public class Cat extends Animal{ }	 // Cat is a Animal
1
2
3

那么我们可以认为狗和猫都是一种特殊的动物，那么可以使用动物类型来表示狗或猫。

Animal a;           // a表示一种动物(用a来存储一种动物)
a = new Dog();      // a的具体类型(真实类型)是狗狗
a = new Cat();      // a的具体类型(真实类型)是猫猫
1
2
3

此时对象（a）具有两种类型:

  • 编译时类型：声明对象变量的类型——>Animal

  • 运行时类型：对象的真实类型 ——>new Dog

当编译时类型和运行时类型不一致的时候，此时多态就产生了:

注意：编译时类型必须是运行类型的父类或接口。

所谓多态 ，简单地理解 , 一个对象在不同时刻具有多种形态。

Animal	a = null;
1

我家有一种动物，你猜它的叫声是怎么样的，猜不到( 为什么？ )，因为这个动物具体是什么不知道，实际可以有多种形态。

  • 如果该动物是狗，叫声是：旺旺旺...

  • 如果该动物是猫，叫声是：妙妙妙...

a = new Dog();	// a此时表示Dog类型的形态
a.shout();		// 旺旺旺...

a = new Cat();	// a此时表示Cat类型的形态
a.shout();		// 喵喵喵...
1
2
3
4
5

说的具体点：同一引用类型的变量，由于引用的真实对象不同，调用同名方法时，方法产生的结果不同。

多态的前提：

  • 有继承/实现关系。可以是继承关系（父类和子类），也可以是实现关系（接口和实现类）
  • 有方法重写
  • 编译时类型的变量引用了运行时类型的对象

注意：多态要求编译时类型必须是运行类型的父类或接口，所以多态操作有两种定义语法：

  • 操作继承关系（开发中不是很多）：
父类 变量 = new 子类();
变量.方法();
1
2
  • 操作实现关系（开发中最频繁）：
接口 变量 = new 实现类();
变量.方法();
1
2

通过以上两种方式创建出来的对象，我们习惯上把它称为多态对象。

# 3.3 多态优缺点

  • 多态的好处: 提高代码的扩展性 , 灵活性

# 3.4 多态的转型 ( 了解 )

  • 自动类型转型 : 把子类类型数据转成父类类型数据 Animal a = new Cat();
  • 强制类型转型 : 把父类类型数据转成子类类型数据 Cat cat = (Cat)a;
public class PolyDemo {

	public static void playWith(Animal a) {
        a.shout();
        // 问题：多态虽好，但不能通过多态对象a调用子类/实现类特有的方法
        // 解决方案: 类型转换
        if( a instanceof Cat ) {
            Cat cat = (Cat)a;
            cat.catchMouse();
        } else if (a instanceof Dog) {
            Dog dog = (Dog)a;
            dog.eatBone();
        }
    }

    public static void main(String[] args) {
        Animal a = new Dog();
        PolyDemo.playWith(a);

        a = new Cat();
        PolyDemo.playWith(a);

        a = new Pig();
        PolyDemo.playWith(a);
    }
}

public class Mouse implements IUSB{

        System.out.println("鼠标移动");

//多态的好处:
可替换性。多态对已存在代码具有可替换性。例如，多态对键盘类工作，对其他键盘，如办公键盘,机械键盘，也同样工作。(需要的是对象的一个功能而不是对象本身)
可扩充性。多态对代码具有可扩充性。增加新的子类不影响已存在类的多态性、继承性，以及其他特性的运行和操作。
接口性。多态是父类通过方法签名，向子类提供了一个共同接口，由子类来实现它。
可维护性和简化性（simplicity）。多态简化对应用软件的代码编写和修改过程，尤其在处理大量对象的运算和操作时，//这个特点尤为突出和重要。
灵活性。它在应用中体现了灵活多样的操作，提高了使用效率。

# 4.2.7 题目7

某公司旗下有三种型号（model）机器人（Roboot） A 、B、 C。

他们都具有姓名name, 型号model的字段 和 走路功能 walk。

其中A 实现了 ISpeak接口中定义的说话功能speak ，B实现了ISee接口中的图像识别功能 see，C 同时实现了说话和识别功能。 请利用继承、接口、多态等知识完成类的设计并编写测试Demo， 创建机器人对象, 并调用实现过的方法。

public class model {
    private String model;
    private String name;
    
    public String getModel() {
        return model;
    }

    public void setModel(String model) {
        this.model = model;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
    public void walk(){
        System.out.println("走路");
    }
}
public class A extends model implements ISpeak{
    
    public void speak() {
        System.out.println("我是A机器人,我会说话");
    }
}
public class B extends model implements ISee {
    
    public void see() {
        System.out.println("我是B机器人,我能人脸识别");
    }
}
public class C extends model implements ISee,ISpeak {
    
    public void see() {
        System.out.println("我是C能看见");
    }

   
    public void speak() {
        System.out.println("我是C能说话");
    }
}
public class RobootDemo {
    public static void main(String[] args) {
        //多态对象的创建
        Model model  = new A();
        model.walk();
        A a = (A)model;
        a.speak();

        //多态接口对象创建
        ISee isee = new C();
        isee.see();
        C c = (C)isee;
        c.speak();
    }
}
//总结:封装+继承+多态(从思想开始)
1.封装思想:将字段和方法放在一个模块中,提高代码重用性,提高安全性.
javaBean规范:
1.类要用public修饰
2.字段私有化,private修饰
3.默认构造器
4.提供设置器和访问器set,get方法

继承:将多个类共有的代码块提取到一个类中,让其他类继承它,继承后,子类拥有父类的字段和方法,提高效率.
语法:class 子类类名 extends 父类类名{}

多态:一个对象具有多种形态.
接口:在多态的情况下,父类会提供一个方法,给子类一个接口去实现,完成解耦,父类就不用再因为子类而新增代码,增加方法等,大大简化了代码.

强制类型转换:把父类类型变量强制转成子类变量,(小转大)
语法: 子类 变量名 = (子类)变量名;
强转后:父类类型变量,编程子类变量,可以调用子类特有方法
注意:多态对象的真实类型必须是子类类型,才能强转.
